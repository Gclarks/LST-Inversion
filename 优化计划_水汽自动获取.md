# 优化计划：MODIS 水汽含量自动获取

## 1. 目标

点击主界面"自动获取"按钮后，根据 Landsat 8 影像的时空信息，自动搜索、下载、裁剪、提取 MODIS 大气水汽产品，将均值水汽含量填入输入框，替代手动输入。

---

## 2. 数据源选择

### 推荐产品：MOD05_L2（Terra / MODIS）

| 属性 | 值 |
|------|-----|
| 产品名称 | MOD05_L2 (MODIS/Terra Water Vapor 5-Min L2 Swath 1km) |
| 卫星 | Terra |
| 过境时间 | ~10:30 AM（与 Landsat 8 几乎同时） |
| 水汽数据层 | `Water_Vapor_Near_Infrared`（近红外反演，仅白天） |
| 空间分辨率 | 1 km |
| 数据格式 | HDF-EOS2 Swath |
| 单位 | cm（= g/cm²，直接可用） |
| 质量层 | `Water_Vapor_Near_Infrared_Quality` |

### 备选产品

| 产品 | 适用场景 |
|------|---------|
| MYD05_L2 (Aqua) | 当 Terra 数据缺失时回退；Aqua ~1:30 PM，间隔较大 |
| MOD07_L2 (Terra) | 夜间影像或 NIR 反演不可用时（含 IR 水汽层） |

> **策略**：优先 MOD05_L2，在同一时间窗口内找不到时回退到 MYD05_L2，再回退到 MOD07_L2。

### 时空匹配约束

- **时间窗口**：Landsat 8 成像时间 ± 1 小时（Terra）；若无结果，放宽到 ± 3 小时
- **空间覆盖**：MODIS 条带必须覆盖 Landsat 8 影像四至范围

## 3. 技术架构

### 3.1 数据获取链路

```
Landsat 8 MTL.txt
    ├─ 成像时间 (DATE_ACQUIRED + SCENE_CENTER_TIME)
    └─ 四至坐标 (CORNER_UL/UR/LL/LR_LAT/LON_PRODUCT)
            │
            ▼
    NASA CMR API 搜索 granules
    GET /search/granules.json?short_name=MOD05_L2&temporal=...&bounding_box=...
            │
            ▼
    取最近匹配的 granule → 获取 download URL
            │
            ▼
    下载 .hdf 文件到临时目录（带 Earthdata 认证）
            │
            ▼
    GDAL 读取 HDF-EOS2 Swath
        - 读取 Water_Vapor_Near_Infrared
        - 读取 Latitude / Longitude 定位数组
            │
            ▼
    空间裁剪至 Landsat 8 四至范围
        - 基于经纬度筛选像素
        - 质量控制：剔除 QA ≠ 0 的像元
            │
            ▼
    计算有效像元均值 → w (g/cm²)
            │
            ▼
    填入 GUI 水汽输入框
```


## 4. 实现细节

### 4.1 CMR 搜索 API

```
GET https://cmr.earthdata.nasa.gov/search/granules.json
Params:
  short_name       = MOD05_L2
  temporal         = 2018-08-15T09:00:00Z,2018-08-15T12:00:00Z
  bounding_box     = 116.65,38.45,118.22,40.13
  page_size        = 10
  sort_key         = -start_date
```

返回 JSON，从中提取：
- `feed.entry[].id` → granule ID
- `feed.entry[].links[].href` → 数据下载 URL（筛选 `title="Download ..."` 的链接）
- `feed.entry[].time_start` / `time_end` → 精确时间

### 4.2 下载认证

NASA Earthdata 要求 HTTP Basic Auth 或 Bearer Token：

```
# 方式一: .netrc（推荐，与 wget/curl 兼容）
machine urs.earthdata.nasa.gov
    login <username>
    password <password>

# 方式二: 环境变量
EARTHDATA_USERNAME=xxx
EARTHDATA_PASSWORD=xxx
```

首次使用时弹出认证窗口，保存到 `~/.netrc`。

### 4.3 HDF 数据处理（GDAL）

```python
# MOD05_L2 内部路径
wv_sds_path = 'HDF4_EOS:EOS_SWATH:"{hdf}":mod05:Water_Vapor_Near_Infrared'
lat_sds_path = 'HDF4_EOS:EOS_SWATH:"{hdf}":mod05:Latitude'
lon_sds_path = 'HDF4_EOS:EOS_SWATH:"{hdf}":mod05:Longitude'
qa_sds_path = 'HDF4_EOS:EOS_SWATH:"{hdf}":mod05:Water_Vapor_Near_Infrared_Quality'

# GDAL 读取
ds = gdal.Open(wv_sds_path)
wv = ds.ReadAsArray()
```

### 4.4 空间裁剪与统计

```
MODIS swath 像素 ──── 遍历 ────
  ├─ lat/lon 落在 Landsat 8 四至范围内？
  │   ├─ 是 → 检查 QA flag
  │   │   ├─ QA = 0（最高质量）→ 纳入统计
  │   │   └─ QA ≠ 0 → 跳过
  │   └─ 否 → 跳过
  │
  └─ 对纳入的 wv 值取均值 / 中位数
```

若有效像素数 < 100（大面积云/水体），标记为低置信度，在日志中警告但仍输出结果。

## 5. 实施步骤

### Step 1：`core/water_vapor.py` — MODIS 水汽获取核心（~150 行）

- `EarthdataAuth` 类：.netrc 读写、token 获取
- `MODISWaterVaporFetcher` 类：
  - `_search_cmr(date, bbox)` → 返回 granule 列表
  - `_pick_best_granule(granules, target_time)` → 选时间最近的产品
  - `_download(href, dest_dir)` → 下载 HDF
  - `_extract_wv(hdf_path, corners)` → 裁剪 + 质量过滤 + 均值
  - `fetch(acq_datetime, corners, temp_dir)` → 一站式接口

### Step 2：`gui/main_window.py` — 按钮启用 + 获取流程

- 启用"自动获取"按钮（从 DISABLED → NORMAL）
- 点击后：
  1. 检查是否已选择输入目录（含 MTL）
  2. 调用 `MODISWaterVaporFetcher.fetch()`
  3. 将返回值填入 `self._wv_var`
  4. 错误处理：弹窗提示

### Step 3：认证弹窗

- 首次使用无 .netrc 时弹出 `EarthdataAuthDialog`
- 或在设置界面新增"MODIS 数据源"标签页

### Step 4：错误处理与回退

- 网络错误 → 提示检查网络，建议手动输入
- 登录失败 → 提示重新输入凭据
- 无覆盖 granule → 提示无匹配 MODIS 数据，建议手动输入
- HDF 读取失败 → 跳过该 granule，尝试下一个

## 7. 风险与边界情况

| 风险 | 缓解措施 |
|------|---------|
| MODIS 数据有延迟（实时影像） | 只有 Reprocessing 数据实时可用；NRT 产品（MOD05_L2_6N）延迟 ~3h。搜索时优先标准产品，回退 NRT |
| Swath 不覆盖研究区 | 放宽时间窗口到 ±3h，或尝试相邻轨道 |
| 研究区大面积云覆盖 | QA 过滤会剔除大部分像素；在日志中警告有效像素偏少 |
| HDF4 驱动不可用 | Windows 下 GDAL 默认含 HDF4；若缺失则提示安装 `gdal-hdf4` |
| 下载速度慢 | 仅下载单景 HDF (~5 MB)，不需分块；设置 60s 超时 |
| Earthdata 认证过期 | Token 过期时重新登录，刷新 .netrc |
